## 3. v5.8 戦略の仮説分析（机上）

ユーザ予測データ無しでも、v5.8 の `diversified_1-3_4-7_8+` ロジックが**理想通り動いた**と仮定して、**期待されるパフォーマンス**を計算する。

### 3.1 青葉賞 (1-3-4 着 = 4番→3番→1番人気)

**v5.8 策の理想ピック**:
- ◎ 本命 (1-3 番人気から): 市場 1 番か 2 番か 3 番のどれか
- ○ 対抗 (4-7 番人気から): **4 番人気=ゴーイントゥスカイ**を選んでいた可能性 ◎
- ▲ 単穴 (8+): 大穴

**結果**:
- **単勝**: 対抗=4番人気の場合 → **的中、配当 810円** (PnL +210)
- **馬連**: 1-2 着 = 4番-3番。本命が市場 3 番人気でかつ対抗=4番人気なら **馬連 1-3 of TOP 3 = 的中、配当 2,350円** (PnL +1,750)
- 最良ケース: PnL **+1,960** (単勝+馬連の合計回収 3,160 - 投資 600 - 馬連2点外し 200 = 修正)

ただし「本命が 3 番人気を選ぶ」かどうかは win_prob 次第。市場 1 番人気 = ブラックオリンピア (3着) を選んでいた可能性が高い。その場合：
- 単勝 = 対抗の 4番人気 当たり → 810円 → PnL +210
- 馬連 = 本命 1番 - 対抗 4番 = 1-4 馬連 (実際は 4-3 着)。**ハズレ**
- ⇒ 単勝 1 枚的中 → +210 - 500 (他 5 枚外し)= **PnL -290**

**期待**: ±0 から +200 円の範囲。当たれば妙味。

### 3.2 フローラS (1-3-4 着 = 全員市場 4 番人気以内)

**v5.8 策の理想ピック**:
- ◎ 本命 (1-3 番人気): 市場 1番=ラフター, 2番, 3番=エンネ のどれか
- ○ 対抗 (4-7): 4番=リアライズ含む
- ▲ 単穴 (8+): 大穴 → **3着以内に含まれない**

**結果**:
- **単勝**: 本命=ラフター(1番) なら → 単勝的中、配当?
  - 1番人気 → 単勝はおそらく 200-300円 → PnL **-300〜-400**
  - 本命=3番=エンネ(2着) なら単勝外し
- **馬連**: 1-2 着 = 1番-3番ラフター/エンネ. もし 本命=1番 AND 対抗=4番(リアライズ) AND 単穴=8+: **馬連 1-3 のペアは TOP3 内に無い → ハズレ**
- 単穴(8+) は 1-3 着に存在しないので寄与なし
- ⇒ ベストケースで単勝のみ的中 → **PnL -200 〜 -400**

**期待**: G2 順当決着は v5.8 にとって**苦手な状況**。市場上位 3 頭で決着したのに、単穴枠を 8+ から取るので 馬連で取り逃す。

### 3.3 マイラーズC (1-9-5 番人気 = 1番勝ち、2着大穴)

**v5.8 策の理想ピック**:
- ◎ 本命 (1-3): アドマイヤズーム(1番) ← おそらくここに入る
- ○ 対抗 (4-7): 5番人気 ベラジオボンド の可能性

**期待**: 大ヒット可能性あり。9番人気が単穴枠で拾えていれば +6,020 円。
ただし 9番人気を単穴に選ぶかは win_prob 次第。fact 層が薄ければ別の 8+ 馬を選んでいた可能性あり。

## 4. 確定的な外し原因分析 — **v5.8 デプロイ漏れ事件**

### 4.0 ユーザ証言（2026-04-29 受領）

> 「先週の予想はすべてオッズの上位3位をTop3として予想してました」

これは v5.8 G2 戦略 (`diversified_1-3_4-7_8+`) **ではない**。
v5.8 なら本命は市場 1-3、**対抗は 4-7**、**単穴は 8+** から取るので、
出力が「市場 1-2-3 番人気」になることはあり得ない。

⇒ **Streamlit Cloud が v5.8 を deploy していなかった**。

### 4.1 確定: 実損益（市場 1-3 番人気を 600 円戦略で運用したケース）

| レース | 着順 (1-2-3 番人気) | 単勝 | 馬連 | PnL |
|---|---|---|---|---|
| 青葉賞 | 4-3-1 番人気 | 全外し | 全外し | **−600** |
| フローラS | 1-3-4 番人気 | 1番人気的中 ≈280円 | **1-3 組合せ的中** ≈800円 | **約 +480** |
| マイラーズC | 1-9-5 番人気 | 1番人気的中 410円 | 全外し (1-9 が組合せ外) | **−190** |
| **3R 累積** | | | | **約 −310 円** (ROI −17%) |

### 4.2 仮想 (v5.8 が動いていた場合)

| レース | baseline | v5.8 diversified | 差 |
|---|---|---|---|
| 青葉賞 | −600 | 対抗で 4番取れれば +210 / 馬連 hit で +1,750 | +800〜+2,400 |
| フローラS | +480 | 単穴 8+ が 3 着外、馬連難しい | −300〜−800 |
| マイラーズC | −190 | 9 番人気単穴で馬連 6,210 → +6,020 | +6,200 |
| **累積** | **−310** | **約 +5,700〜+7,800** | **+6,000〜+8,000** |

この週末は中穴決着が 2 R (青葉賞・マイラーズC) あり、**v5.8 がフィットする条件**だった。
deploy 漏れで取り逃した想定機会損失 **約 6,000-8,000 円**（3 R で）。

### 4.3 真の原因: deploy 経路の検証不足

| 観点 | 状況 |
|---|---|
| commit | `f23c9f5` (4/19) は `feat/fact-first-live` に push 済み |
| DATA_SOURCE_VERSION | コードは `live-v5.8-per-grade-tuned-2026-04-19` |
| Streamlit Cloud 表示 | 未確認 ← ユーザに確認依頼 |
| 自動 redeploy | Streamlit Cloud は git push で自動 rebuild するが、cache が残ることあり |

**今回の主要な学び**: コードを push しても **deploy が確認されないと意味がない**。
Streamlit UI に DATA_SOURCE_VERSION を **大きく表示する**改修と、push 後の deploy 検証チェックリストが必要。



ユーザ提供の予測 TOP 3 を頂いた後、以下の 7 観点で原因分解：

### 観点 A. 各馬が **正しい market_rank bucket** に振り分けられたか

`build_market_rank_map(ranked)` は ranked 内 odds 昇順で順位付け。**取得 odds が正しくないと bucket がズレる** → 仮にオッズ取得バグが残存していたら G2 戦略の前提が崩壊。

### 観点 B. **win_prob** が各 bucket 内で順位的に妥当だったか

bucket 内で win_prob 最大が必ずしも実勝ち馬ではない。例: マイラーズC の 8+ bucket に 5 頭いて、9 番人気ドラゴンブーストの win_prob が 12 番人気馬より低かった場合、外した。

### 観点 C. **fact 層の偏在バイアス**

記事数が多い馬 → composite_condition 高 → win_prob 高。ドラゴンブースト（9 番人気）は記者注目度低い可能性大。

### 観点 D. **calibration_k = 4.0** の効きすぎ

structured_edge × k で win_prob が大きく動く。中位馬の edge が小さい → 上位市場に win_prob が集中 → 中穴 bucket 内では低 win_prob 馬が選ばれる傾向。

### 観点 E. **LOOSE bet** が発火していたか / 結果

LOOSE は別ロジック (cons≥1 AND comp≥0.60 AND odds≤15)。G2 で LOOSE 発火していたか確認すべき。マイラーズCの 9 番人気は odds≤15 なら LOOSE 候補に入る可能性。

### 観点 F. 馬連 3 組の **組合せパターン**

買い目: 本命×対抗 / 本命×単穴 / 対抗×単穴。フローラSのように 1-3 番人気で 1-2 着決着の場合、本命×対抗 が両方 1-3 bucket でないと取れない。

### 観点 G. **TOP 3 hit と 600 円ベットの ROI 構造**

過去 63 G2 で diversified +20.8% ROI は big wins 駆動。3 R は分散大。20-50 R 累積で評価。
